MAD Language Documentation

Michigan Algorithm Decoder (MAD) is an early, highly influential algebraic programming language developed in 1959. This documentation covers the modern MAD IDE pipeline, syntax rules, and original historical code examples.

Basic Syntax & Formatting

Original MAD programs required strict columnar formatting for IBM 7090 execution. The MAD IDE intelligently processes statements so you can write freely, but the underlying transpiler relies on standard conventions.

Note: Labels are historically restricted to 6 characters and padded to column 11. MAD IDE automatically pads line labels to column 11 for you. 
           NORMAL MODE IS INTEGER
           DIMENSION KEY(32), MYTRAN(4)
           INITAS.(0)
           PRINT COMMENT $WHICH SCRIPT DO YOU WISH TO PLAY$

CTSS Abbreviations

The Compatible Time-Sharing System (CTSS) introduced abbreviations to speed up coding on teletypes. Our IDE automatically expands these at compile time:

AbbreviationExpansion
W'RWHENEVER
O'EOTHERWISE
E'LEND OF CONDITIONAL
T'HTHROUGH
T'OTRANSFER TO
F'NFUNCTION RETURN
E'NEND OF FUNCTION

How to Run MAD Code

To run any MAD code inside the MAD IDE, follow these steps:

  1. Open your .mad file using File -> Open, or create a new file and paste your code.
  2. Make sure you have built the underlying compiler first by clicking Tools -> Build MAD Compiler.
  3. Click the ▶ Run button in the top toolbar or press F5.
  4. The IDE will automatically apply CTSS pre-processing, compile the code natively to C, and execute the binary!

Original MAD Code: External Functions

MAD supports modularity via external functions. Variables are typically passed by reference, and arrays are dimensioned using the DIMENSION keyword.

           EXTERNAL FUNCTION (KEY, MYTRAN)
           NORMAL MODE IS INTEGER
           ENTRY TO CHANGE.
           LIST.(INPUT)
           VECTOR VALUES G(1) = $TYPE$, $SUBST$, $APPEND$, $ADDS,
           $START$, $RANKS, $DISPLAS
           VECTOR VALUES SNUMB$ = $13 $
           FIT=0
CHANGE     PRINT COMMENT $ PLEASE INSTRUCT ME$
           LISTRD.(MTLIST.(INPUT),0)
           JOB = POPTOP.(INPUT)
           THROUGH IDENT, FOR J = 1,1, J .G. 7
IDENT      WHENEVER G(J) .E. JOB, TRANSFER TO THEMA
           PRINT COMMENT $CHANGE NOT RECOGNIZED$
           TRANSFER TO CHANGE
THEMA      WHENEVER J .E. 5
           FUNCTION RETURN IRALST.(INPUT)
